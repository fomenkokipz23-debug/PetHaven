# Посібник розробника системи PetHaven

Цей документ призначений для інженерів, які займаються підтримкою та розширенням архітектури проєкту.

##  Архітектурний дизайн (Clean Architecture)
Проєкт розділено на слабкопов'язані шари, де залежності спрямовані всередину ядра предметної області:
`Domain` (Ядро) <- `Application` <- (Сервіси) <-  `Infrastructure` (Збереження JSON) / `Console` (UI).

##  Реалізація SOLID-принципів у коді
* **Single Responsibility Principle (SRP):** Сутність `Booking` відповідає суто за стан конкретного проживання. Сервіс `BookingService` оркеструє бізнес-процес, а клас `JsonFileStore<T>` інкапсулює роботу із серіалізацією `System.Text.Json`.
* **Open/Closed Principle (OCP):** Розрахунок цін реалізовано через патерн **Strategy** (`IPricingStrategy`). Додавання нових гнучких тарифів (наприклад, "Святковий") здійснюється створенням нового класу без модифікації існуючих доменних моделей.
* **Dependency Inversion Principle (DIP):** Шар сервісів (`Application`) не знає про файлову систему комп'ютера. Він працює виключно через абстрактні інтерфейси `IBookingRepository` та `IRoomRepository`, визначені в шарі `Domain`.

##  Правила розширення системи
Щоб додати нову стратегію розрахунку цін, створіть новий клас у шарі `PetHaven.Domain`, який реалізує інтерфейс `IPricingStrategy`, та реалізуйте метод `CalculatePrice`. Поточний код сутності `Booking` змінювати не потрібно.
