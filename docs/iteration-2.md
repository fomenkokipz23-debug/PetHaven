# Журнал виконання Ітерації 2 (Лабораторна робота №35)

## 1. Реалізовані бізнес-правила (Invariants & Rules)
1. **Диференціація цін (Strategy):** озрахунок вартості проживання враховує вид тварини (коти мають 10% знижку, собаки - 10% націнку за вигул).
2. **Контроль станів:** скасувати або завершити можна тільки активне бронювання. Повторна зміна статусу викликає `InvalidOperationException`.
3. **Звільнення ресурсів:** при скасуванні чи завершенні проживання кімната автоматично отримує статус `IsOccupied = false`.
4. **Валідація часу заїзду:** заборонено створювати бронювання «заднім числом».

## 2. Патерн розширення
Обрано патерн **Strategy** (`IPricingStrategy`). 
* **Чому:** він дозволяє додавати нові фінансові тарифи (наприклад, "Святковий", "Вікенд", "Довготривала оренда") шляхом створення нового класу, абсолютно не змінюючи код сутності `Booking`.
* **Що було б без нього:** довелося б писати заплутані конструкції `if/else` або `switch` всередині сутності домену, порушуючи Open/Closed Principle (принцип відкритості/закритості SOLID).

## 3. Об'єкт аналітики та LINQ
Реалізовано `AnalyticsService`, який за допомогою LINQ-запитів рахує загальний прибуток (`Sum`), фільтрує активні бронювання (`Where`), групує тварин через `GroupBy` та трансформує дані в оптимізовану колекцію `Dictionary<PetType, int>`.
