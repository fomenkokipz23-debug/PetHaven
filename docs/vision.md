# Vision: Система керування готелем для тварин "PetHaven"

## 1. Опис проблеми
Власники готелів для тварин стикаються з проблемою хаотичного обліку постояльців, овербукінгу (бронювання одного вольєра двічі) та відсутності швидкого доступу до специфічних вимог догляду за твариною (раціон, ліки). "PetHaven" автоматизує процес бронювання, контролює зайнятість місць та зберігає профілі тварин.

## 2. Цільові користувачі
* **Адміністратор готелю** - реєструє тварин, оформлює заселення/виселення, перевіряє наявність вільних місць.

## 3. Основні сценарії використання (Use Cases)
1. **Заселення тварини (Check-In):** Адміністратор створює профіль тварини та власника, вибирає відповідний вільний вольєр і фіксує дату заїзду/виїзду.
2. **Перегляд доступних вольєрів:** Пошук вільних кімнат за типом тварини (кішка/собака) та розміром.
3. **Розрахунок вартості проживання:** Автоматичне обчислення фінальної суми на основі тарифу вольєра та тривалості перебування.

## 4. Нефункціональні вимоги
* **Тестованість (Testability):** Бізнес-логіка ізольована від UI та збереження даних, що дозволяє покрити її юніт-тестами на 100%.
* **Надійність (Invariant Protection):** Об'єкти системи (наприклад, Бронювання) не можуть бути створені в некоректному стані (наприклад, дата виїзду раніше дати заїзду).
* **Простота розширення:** Можливість легко замінити збереження в пам'яті (In-Memory) на реальну БД на наступних ітераціях.

## 5. Обмеження першої ітерації
* Дані зберігаються виключно в оперативній пам'яті (In-Memory) під час роботи програми.
* Сценарії скасування бронювання, оплати карткою та генерації звітів переносяться на наступні ітерації.
