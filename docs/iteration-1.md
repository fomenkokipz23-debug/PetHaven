# Звіт про виконання Ітерації 1 (Lab 34)

## Що виконано й працює
1. Повністю створено каркас застосунку відповідно до вимог розділення шарів (Domain -> Application -> Infrastructure -> UI).
2. Реалізовано 3 основні класи предметної області (`Pet`, `Room`, `Booking`).
3. Забезпечено валідацію вхідних даних у конструкторах для захисту бізнес-правил.
4. Створено працюючий вертикальний зріз: від консольного меню через сервіс до збереження об'єктів в `InMemory`-репозиторіях.
5. Написано юніт-тести для перевірки розрахунку вартості заїзду.
6. Налаштовано автоматичний GitHub Actions workflow.

## Класи, підготовлені під розширення
* Інтерфейси `IRoomRepository` та `IBookingRepository` готові для того, щоб на Ітерації 2 (Lab 35) замінити In-Memory колекції на збереження даних у JSON-файли без зміни логіки `BookingService`.
