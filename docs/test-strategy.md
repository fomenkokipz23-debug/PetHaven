# Тестова стратегія готелю для тварин "PetHaven"

## 1. Критичні бізнес-сценарії для перевірки
* **Заселення (Check-In) з бізнес-правилами:** Перевірка конфліктів завантаженості кімнат, застосування знижок/націнок через `IPricingStrategy` залежно від типу тварини.
* **Скасування та виселення:** Контроль коректності переходів між статусами (Active -> Completed/Cancelled) та обов'язкове асинхронне вивільнення номерного фонду.
* **Стійкість Persistence-шару:** Поведінка системи при відсутності файлів даних, порожніх файлах або пошкодженому JSON-форматі.

## 2. Зони ризику та архітектурні шви (Seams)
* **Поділ відповідальностей:** Шар бізнес-логіки повністю абстрагований від дисків за допомогою інтерфейсів `IBookingRepository` та `IRoomRepository` (DIP).
* **Застосування ізоляції:** Для юніт-тестів сервісів ми використовуємо `Moq` для підміни репозиторіїв. Для інтеграційних тестів використовуємо реальний `JsonFileStore<T>`, що працює у тимчасових папках ОС (`Path.GetTempFileName`).

## 3. Негативні сценарії (Fault Handling)
* Повторне скасування вже скасованого або завершеного проживання (захист від подвійних операцій).
* Спроба заселення тварини у кімнату, яка вже зайнята.
* Обробка винятків при пошкодженні або блокуванні JSON-файлу на диску.
