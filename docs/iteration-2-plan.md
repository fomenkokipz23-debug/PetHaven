# План Ітерації 2 (Лабораторна робота №35) - PetHaven

## 1. Сценарії з Backlog у реалізацію
* **UC-1: Заселення тварини (Check-In)** - розширення валідації конфліктів кімнат.
* **UC-2: Виселення тварини (Check-Out)** - звільнення кімнати, фінальний розрахунок вартості з урахуванням стратегії знижок/націнок.
* **UC-3: Скасування бронювання** - видалення запису або зміна статусу з поверненням кімнати в доступні.
* **UC-4: Аналітичні запити** - фільтрація, сортування та статистика завантаженості готелю.

## 2. Статус класів та точки розширення
* **Без змін:** Класи `Pet` та `Room` залишаються майже без змін.
* **Зміни:** `Booking` отримує статус (Active, Completed, Cancelled).
* **Точка розширення:** Впровадження патерну **Strategy** (`IPricingStrategy`) для гнучкого нарахування вартості проживання без зміни коду сутності `Booking`.

## 3. Ризики
* Складність серіалізації зв'язаних об'єктів (циклічні посилання при збереженні `Booking -> Pet/Room`). Вирішується збереженням ідентифікаторів (ID) або налаштуванням `ReferenceHandler`.
